  • The useradd command is a low-level utility.
  • It is used for creating user accounts in Linux.
  • The adduser command can also be used, it is a symbolic link to useradd.
Note

  • The ln utility is used to create a new directory entry/linked file.
  • This is useful for maintaining multiple copies of a file in many places at once without using up storage.
  • There are two types of links known as HARD link and SYMBOLIC link.
  • By default, the ln command creates a hard link, you will need to apply the option -s to create a symbolic link.
Note
  •  
  • Hard link vs symbolic link
  •  
  • An inode is a data structure in a linux file system which describes a filesystem object such as a file or directory.
  • A file will hold a link an inode.
  • A hard link will hold a link to the same inode as the file.
  • A symbolic link will hold a link to the file that holds a link to the inode.

  • The rm command is commonly used to delete files.
  • It can also be used for a variety of other deletion needs, such as links see below.
Note
  •  
  • USE WITH CAUTION
  •  
  • rm -f filename/linkname     will forcefully remove the file/link.
  • rm -r directoryName              will recursively remove files in the working directory.
  • rm -rf directoryName            will remove the specified directory and all files and subdirectories from within.
  • rm -i filename                          will request confirmation prior to deletion.
  •  
  • The rmdir command is used to delete empty directories.
Note
  •  
  • Please use the rm -rf directoryName command to delete non-empty directories.
  •  
  • The userdel command is used to delete user accounts from linux systems.
Note
  •  
  • The addition of the option -r on userdel will cause all files in the user's home directory will be removed along with home directory itself and the user's mail spool.

    Files located in other file systems will have to be searched for and deleted manually.

  •  
  • The groupdel command is used to delete groups from linux systems.
  • Used by root to remove a group.
  • You may not remove the primary group of any existing user, you must remove the user before you remove that user's primary group.
Note
  •  
  • groupdel exits with the following value
  •  
  • 0       -     Success
  • 3       -     Invalid command syntax
  • 6       -     Specified group doesn't exist
  • 8       -     Can't remove user's primary group
  • 10     -     Can't update group file

  • The mv command is commonly used to move or rename files and directories in a linux system.
Note
  • mv      {oldFilename/source}   {newFilename/dest}     [Standard file rename]
  • mv -v   {oldFilename/source}   {newFilename/dest}     [Verbose output]
  • mv -i   {oldFilename/source}   {newFilename/dest}     [Interactive pass, user prompt]

  • The vi command is used to edit the contents of files in a linux system.
  • It is short for "vi"sual editor.
  • It has two modes COMMAND and INSERT mode
    • Command mode, letters you type perform functions.
    • Insert mode, letters you type forms words and sentences.
    • To exit insert mode press the Esc key.
  • A more feature rich version of vi is vim, it stands for vi improved..
  • An easier alternative to vi or vim is nano.
  • Enter command man nano to see how to use this editor.

  • vi filename will start the vi editor in command mode, to enter insert mode type 'i'.
  • To quit insert mode type Ctrl+c or press the Esc key.
  • To undo any changes type 'u' when in command mode, to undo all changes type 'U'.
Note
  •  
  • Exit
  • :x        Exits vi, saving changes.
  • :q        Quits vi as long as no changes have been made, add ! to override.
  • :q!       Quits vi and ignores any changes.
  • :wq     Writes changes and quits vi.
  • ZZ       Exits vi, saving changes.
  •  
  • Navigation
  • Arrow key or h     Navigate left.
  • Arrow key or l      Navigate right.
  • Arrow key or k     Navigate up.
  • Arrow key or j      Navigate down.
  • H     Jump to top.
  • L     Jump to bottom.
  • /     Starts search, provide keyword. Use n to navigate through matches.

  • The touch command paired with the option -a is used to change file/directory accessed times.
  • By default touch -a fileName/directoryName will use the current system time.
Note
  •  
  • You can explicitly specify the time using options -t or -d, see below:
  •  
  • The format for specifying -t is [[CC]YY]MMDDhhmm[.SS]
  • The format for specifying -d is '2017-01-14 12:12:12.000000000 +0530'
  •  
  • CC      -      Specifies the first two digits of the year (the century)
  • YY      -      Specifies the last two digits of the year.
  • MM      -      Specifies the month, from 01 to 12
  • DD      -      Specifies the day, from 01 to 31
  • hh      -      Specifies the hour, from 00 to 23
  • mm      -      Specifies the minute, from 00 to 59
  • SS      -      Specifies the seconds, from 00 to 61
  •  
  • If the CC and YY letter pairs are not specified, the value defaults to the current year.
  • If the SS letter pair is not specified, the value defaults to 0.
  •  
  • You can also copy the timestamp from another file/directory using the command: touch {source:fileName/directoryName} -r {dest:fileName/directoryName}
  • The touch command paired with the option -m is used to change file/directory modified times.

  • The usermod command paired with the option -l is used to rename linux user accounts.

  • The passwd command is used to reset the password of a users account in linux.
Note
  •  
  • The passwd command is used to reset the password of user accounts in linux.
  • The following options will change the way the passwd command operates, see below:
  •  
  • -d username         Deletes a users password, this is a quick way to disable logins for an account (without disabling the account itself)
  • -e username         Immediately expire an accounts password, this forces a user to change their password the next time they login.
  • -i X username    This option is used to disable an account after the password has been expired for X number of days.
  • -l username         Lock the password of the named account, meaning they are not allowed to change their password.
  • -n X username    Set the minimum number of days X between password changes, 0 means the user may change their password at any time.
  • -q username         Quiet mode, will operate without displaying any output.
  • -S username         Displays account status information. Specifying the option -a in addition to -S will display password status for all users.
  • -u username         Unlock the password of the named account.
  • -w X username    Set the number of days X of warning, before a password change is required.
  • -x X username    Set the maximum number of days X a password remains valid
  •  
  • The usermod command paired with the option -u is used to assign a new UID to linux user accounts.
  • Use id loginName command in order to verify the new UID has been successfully assigned.

  • The cat command is used to read data from files and output their contents.
  • It is the simplest way to display the contents of a file in a linux system.
Note
  •  
  • The following options will change the way the cat command operates, see below:
  • -n file         Prints contents of file, including line numbers
  • -s file         Prints contents of file, omitting blank lines.
  •  
  • Additionally:
  • cat file1 file2                Prints contents of both file1 and file2.
  • cat file1 > file2           Reads the content of file1 and writes it to the file2 overwriting existing content. If file2 doesn't exist it will be created.
  • cat file1 >> file2         Reads the content of file1 and appends to the end of file2, if file2 doesn't exist it will be created.
  •  
  • The stat command is used to display the detailed status of a particular file or directory.
  • The ls command is used to list all directory contents.
Note
  •  
  • The following options will change the way the ls command operates, see below:
  • -a               List all files including hidden file starting with '.'.
  • -d */        List all directories.
  • -i               List contents including inode numbers.
  • -l               List with long format including permissions.
  • -la             List with long format including hidden files.
  • -lh             List with long format including readable file size.
  • -ls             List with long format including file size.
  • -r               List in reverse order. (Alphabetical)
  • -R               List directory tree recursively.
  • -s               List inluding file size.
  • -S               List sorted by file size.
  • -t               List sorted by time and date.
  • -X               List sorted by extension name.
  •  
  • Additionally:
  • ls *.txt             List only text miles with wildcard .txt.
  • ls -d $PWD/*    List with full path.
