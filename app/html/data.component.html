• The touch command is commonly used to create new empty files.
  • It can also be used to update a files last modification or accessed date, please see:
    • Modify > File > Modified/Accessed.
  • The nano command can also be used to create new files e.g.
    • nano newFile.txt

  • The mkdir command is used to create new directories.
  • A directory, referred to as a folder in some operating systems.
  • It appears to the user as a container for other directories and files.
  • The useradd command is a low-level utility.
  • It is used for creating user accounts in Linux.
  • The adduser command can also be used, it is a symbolic link to useradd.
Note
  •  
  • useradd -d home/directory username     will create a user with a home/directory.
  • useradd -g groupname username                 will create a new user and assign them a group.
  •  
  • The groupadd command is used to add user groups to a linux system.
Note
  •  
  • groupadd exits with the following value
  •  
  • 0     -   Success
  • 2     -   Invalid command syntax
  • 3     -   Invalid argument to option
  • 4     -   GID not unique
  • 9     -   Group name not unique
  • 10   -   Can't update group file

  • The ln utility is used to create a new directory entry/linked file.
  • This is useful for maintaining multiple copies of a file in many places at once without using up storage.
  • There are two types of links known as HARD link and SYMBOLIC link.
  • By default, the ln command creates a hard link, you will need to apply the option -s to create a symbolic link.
Note
  •  
  • Hard link vs symbolic link
  •  
  • An inode is a data structure in a linux file system which describes a filesystem object such as a file or directory.
  • A file will hold a link an inode.
  • A hard link will hold a link to the same inode as the file.
  • A symbolic link will hold a link to the file that holds a link to the inode.
  •  
  • The rm command is commonly used to delete files.
  • It can also be used for a variety of other deletion needs, such as links see below.
Note
  •  
  • USE WITH CAUTION
  •  
  • rm -f filename/linkname     will forcefully remove the file/link.
  • rm -r directoryName              will recursively remove files in the working directory.
  • rm -rf directoryName            will remove the specified directory and all files and subdirectories from within.
  • rm -i filename                          will request confirmation prior to deletion.
  •  
  • The rmdir command is used to delete empty directories.
Note
  •  
  • Please use the rm -rf directoryName command to delete non-empty directories.
  •  
  • The userdel command is used to delete user accounts from linux systems.
Note
  •  
  • The addition of the option -r on userdel will cause all files in the user's home directory will be removed along with home directory itself and the user's mail spool.

    Files located in other file systems will have to be searched for and deleted manually.

  •  
  • The groupdel command is used to delete groups from linux systems.
  • Used by root to remove a group.
  • You may not remove the primary group of any existing user, you must remove the user before you remove that user's primary group.
Note
  •  
  • groupdel exits with the following value
  •  
  • 0     -   Success
  • 3     -   Invalid command syntax
  • 6     -   Specified group doesn't exist
  • 8     -   Can't remove user's primary group
  • 10   -   Can't update group file

  • The mv command is used to move or rename files.
Note
  • mv        old-filename/source   new-filename/dest     [Standard file rename]
  • mv -v   old-filename/source   new-filename/dest     [Verbose output]
  • mv -i    old-filename/source   new-filename/dest     [Interactive pass, user prompt]
  • mv -f   old-filename/source   new-filename/dest      [Force]
  • The vi command is used to edit the contents of files.
  • It is short for "vi"sual editor.
  • It has two modes COMMAND and INSERT mode:
    • Command mode: letters you type perform functions.
    • Insert mode: letters you type form words and sentences.
  • A more feature rich version of vi is vim, it stands for vi improved.
  • An easier alternative to vi or vim is nano, man nano.

  • vi will start in command mode, to enter insert mode type 'i'.
  • To quit insert mode type Ctrl+c or press the Esc key.
  • To undo any changes enter command mode and type 'u', to undo all changes type 'U'.
Note
  •  
  • Exit
  • :x        Exits vi, saving changes.
  • :q        Quits vi as long as no changes have been made, add ! to override.
  • :q!       Quits vi and ignores any changes.
  • :wq     Writes changes and quits vi.
  • ZZ       Exits vi, saving changes.

  • Navigation
  • Arrow key or h     Navigate left.
  • Arrow key or l      Navigate right.
  • Arrow key or k     Navigate up.
  • Arrow key or j      Navigate down.
  • H     Jump to top.
  • L     Jump to bottom.
  • /     Starts search, provide keyword. Use n to navigate through matches.

  • The passwd command is used to reset the password of a users account in linux.
Note
  • The following commands will change the way the passwd command operates.
  •  
  • -d username        Deletes a users password, this is a quick way to disable logins for an account (without disabling the account itself).
  • -e username        Immediately expire an accounts password, this forces a user to change their password the next time they login.
  • -h                            Display information about how to use the passwd command.
  • -i x username      This option is used to disable an account after the password has been expired for (x) number of days.
  • -l username         Lock the password of the named account, meaning they are not allowed to change their password.
  • -n x username     Set the minimum number of days (x) between password changes, 0 means the user may change their password at any time.
  • -q username        Quiet mode, will operate without displaying any output.
  • -S username        Displays account status information, specifying -a in addition to -S will display password status for all users.
  • -u username        Unlock the password of the named account.
  • -w x username     Set the number of days (x) of warning before a password change is required.
  • -x x username      Set the maximum number of days (x) a password remains valid
